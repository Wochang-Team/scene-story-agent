# LLM Wiki 작업 규칙

이 문서는 LLM Wiki를 운영할 때 따르는 최소 실행 규칙이다.

## 기본 원칙

위키는 개발 중 얻은 외부 자료와 인사이트를 작게 누적한다.

- Git으로 관리되는 프로젝트 내부 파일은 `.wiki/raw/`에 복사하지 않는다.
- 프로젝트 내부 파일은 원본 경로를 출처로 직접 연결한다.
- `.wiki/raw/`는 웹 문서, PDF, 회의록, 캡처, 외부 조사 자료처럼 Git 밖에서 들어온 자료만 보관한다.
- LLM은 `.wiki/raw/` 안의 원문을 수정하지 않는다.
- LLM이 작성하고 수정하는 문서는 `.wiki/index.md`, `.wiki/log.md`, `.wiki/insights.md`로 한정한다.
- 새 자료가 기존 내용과 충돌하면 기존 문서를 조용히 덮어쓰지 않고 충돌 내용을 기록한다.
- LLM Wiki 운영 안내는 이 문서에 둔다.

## 디렉터리

위키는 아래 구조만 사용한다.

- `.wiki/raw/`: 외부 원문 자료
- `.wiki/raw/assets/`: 외부 원문에 딸린 이미지, PDF, 첨부 파일
- `.wiki/index.md`: 전체 색인
- `.wiki/log.md`: 작업 이력
- `.wiki/insights.md`: 프로젝트를 통해 얻은 공유 가능한 인사이트

## 문서 작성 기준

문서는 빠르게 읽고 바로 적용할 수 있게 쓴다.

- 제목은 문서의 대상을 바로 드러낸다.
- 첫 문장은 문서의 목적이나 결론을 짧게 쓴다.
- 핵심 내용은 항목명으로 나눈다.
- 코드명, API명, 제품명, 외부 서비스명은 원문 표기를 유지한다.
- 불확실한 내용은 단정하지 않고 `확인 필요`로 표시한다.
- 민감정보, 비밀값, 개인 데이터는 기록하지 않는다.
- 문서 하단에는 `## 이력관리`를 둔다.

## 출처 표기

출처는 추적 가능해야 한다.

- 프로젝트 내부 파일은 실제 경로를 적는다.
- 외부 원문 파일은 `.wiki/raw/` 경로를 적는다.
- 웹 출처는 URL과 확인일을 적는다.
- 출처 없는 추론은 `확인 필요`로 분리한다.

## Ingest 절차

새 외부 원문은 하나씩 처리한다.

- 1단계: `.wiki/raw/`에 외부 원문을 보관한다.
- 2단계: `.wiki/index.md`에서 기존 관련 항목을 확인한다.
- 3단계: 필요한 내용만 `.wiki/index.md` 또는 `.wiki/insights.md`에 반영한다.
- 4단계: 기존 내용과 충돌하면 `.wiki/log.md`에 충돌 내용을 기록한다.
- 5단계: 처리 결과를 `.wiki/log.md`에 추가한다.

## Query 절차

질문은 프로젝트 파일과 위키를 함께 기준으로 답한다.

- 1단계: `.wiki/index.md`에서 관련 항목을 찾는다.
- 2단계: 필요한 프로젝트 원본 파일 또는 `.wiki/raw/` 자료를 확인한다.
- 3단계: 답변에 사용한 근거 경로를 표시한다.
- 4단계: 공유 가능한 배움이 있으면 `.wiki/insights.md`에 남긴다.

## Lint 절차

위키 상태는 주기적으로 점검한다.

- Git 관리 파일이 `.wiki/raw/`에 복사되어 있지 않은지 확인한다.
- `.wiki/index.md` 링크가 실제 파일을 가리키는지 확인한다.
- `.wiki/log.md`에 주요 작업 이력이 남아 있는지 확인한다.
- `.wiki/insights.md`에 프로젝트 특화 비밀값이나 민감정보가 없는지 확인한다.
- 오래된 주장이나 확인되지 않은 외부 자료가 있으면 표시한다.
