# scene-story-agent 프로젝트 초안

## 1. 프로젝트 개요

### 프로젝트명
**scene-story-agent**

### 한 줄 설명
사용자가 현장에서 촬영한 사진과 짧은 메모를 바탕으로 장면(Scene)을 해석하고, 이를 시간축 위의 이야기(Story)로 구성하여 이후 유사한 상황이나 재방문 시 개인화된 회상과 추천을 제공하는 멀티모달 AI 에이전트

### 프로젝트 정의
이 프로젝트는 단순히 사진을 저장하는 앱이 아니라, 사진·영상·메모를 하나의 장면 단위로 이해하고, 여러 장면을 연결해 사용자만의 경험 스토리를 축적하는 **기억형 AI 에이전트**를 만드는 것을 목표로 한다.

사용자가 현장에서 남긴 장면은 이후 장기 기억으로 저장되며, 새로운 입력이 들어오면 과거 기억과 연결되어 다음과 같은 경험을 제공한다.
- 이전에 방문했던 장소인지 회상
- 과거에 무엇을 먹었는지 또는 어떤 경험을 했는지 요약
- 당시 남긴 감정과 메모를 바탕으로 맥락 제공
- 누적 기록을 기반으로 개인 취향과 패턴 분석

---

## 2. 문제 정의

사용자는 일상 속에서 다양한 사진과 짧은 기록을 남기지만, 시간이 지나면 그 순간의 맥락을 잃기 쉽다.

기존 사진 앱은 다음 한계가 있다.
- 사진을 파일 단위로만 저장한다.
- 같은 시간대의 여러 사진을 하나의 경험으로 묶지 못한다.
- 과거 기록을 현재 상황과 연결하지 못한다.
- 장기적으로 누적된 개인 취향이나 패턴을 읽어내지 못한다.

따라서 필요한 것은 단순 저장 시스템이 아니라,
**장면을 사건으로 구조화하고, 이를 이야기 단위로 연결하며, 다시 꺼내 쓸 수 있는 기억형 AI 에이전트**이다.

---

## 3. 프로젝트 목표

### 핵심 목표
- 사용자의 사진·영상·메모를 하나의 **Scene**으로 해석한다.
- 관련 Scene들을 하나의 경험 이벤트로 묶어 **Story**로 저장한다.
- 저장된 Story를 장기 기억으로 관리한다.
- 새로운 입력이 들어왔을 때 과거 Story를 조회해 현재 상황에 맞는 회상·비교·추천을 제공한다.

### 기대 효과
- 사용자는 단순히 사진을 보관하는 것이 아니라, 자신의 경험을 서사적으로 축적할 수 있다.
- 같은 장소, 비슷한 메뉴, 유사한 분위기에서의 과거 선택을 다시 활용할 수 있다.
- 시간이 지날수록 개인화된 취향 분석과 추천이 가능해진다.

## 4. 핵심 시나리오

### 시나리오 1. 첫 장면 기록
1. 사용자가 현장에서 사진을 찍는다.
2. 시스템이 사진을 분석해 장면의 성격을 파악한다.
   - 간판
   - 메뉴판
   - 음식
   - 영수증
   - 공간 분위기
3. 같은 시간대의 사진과 메모를 하나의 Scene 묶음으로 정리한다.
4. Scene들을 하나의 Story로 구성해 저장한다.

**예시**
- 장소: 성수동 OO식당
- 메뉴: 돈카츠, 냉모밀
- 금액: 27,000원
- 메모: 웨이팅 길었지만 만족도 높음
- 감정: 만족

### 시나리오 2. 재방문/재경험 인식
1. 사용자가 같은 장소에서 다시 사진을 찍는다.
2. 시스템이 장면을 분석하고 기존 Story와 비교한다.
3. 같은 장소 또는 유사한 Scene으로 판단되면 과거 Story를 조회한다.
4. 이전 방문 이력과 현재 맥락을 결합해 응답한다.

**예시 응답**
- 여기는 2번째 방문이에요.
- 지난번에는 차슈라멘을 먹었어요.
- 메모에는 “국물은 진하지만 조금 짰음”이라고 남겼어요.
- 이번에는 아직 안 먹은 메뉴를 시도해볼 수 있어요.

### 시나리오 3. 누적된 이야기 기반 개인화
1. Scene과 Story가 반복적으로 축적된다.
2. 시스템이 사용자의 취향과 패턴을 추론한다.
3. 이후 새로운 Scene이 들어오면 단순 조회가 아니라 개인화된 해석을 제공한다.

**예시**
- 최근 한 달간 가장 자주 방문한 카테고리는 일식
- 혼자 방문할 때는 면류 선택 비율이 높음
- 2만원대 식당에서 만족도가 가장 높음

## 5. AI 에이전트 관점의 정의

이 프로젝트는 단순 앱이 아니라 다음 흐름을 가진 AI 에이전트이다.

**입력 → 해석 → 이벤트화 → 기억 저장 → 기억 조회 → 판단 → 응답 → 기억 갱신**

### 앱과의 차이
일반 앱은 보통 다음 구조를 가진다.
- 입력
- 저장
- 조회
- 출력

반면 이 프로젝트는 다음을 수행한다.
- 장면의 의미를 해석한다.
- 여러 입력을 하나의 사건으로 묶는다.
- 과거 기억을 조회한다.
- 현재 상황에 맞는 응답 방식을 결정한다.
- 결과를 다시 장기 기억에 반영한다.

즉, 이 시스템의 핵심은 **사진 저장 기능**이 아니라,
**장면을 이야기로 구성하고, 기억을 불러와 현재 상황에 개입하는 것**이다.

## 6. 에이전트 동작 흐름

### 1) 입력 수집
- 사용자가 사진 또는 짧은 영상을 업로드
- 필요 시 한 줄 메모, 감정, 만족도 추가

### 2) 장면 해석
- Vision 모델이 입력을 분석
- 사진/영상의 종류와 핵심 정보 추출
- 가게명, 메뉴명, 금액, 분위기, 카테고리 추론

### 3) Scene 구성
- 같은 시간대의 입력을 하나의 Scene 묶음으로 정리
- 동일한 방문 또는 동일한 경험 맥락으로 그룹화

### 4) Story 생성
- 여러 Scene 정보를 조합해 하나의 경험 Story 생성
- 장소, 메뉴, 금액, 감정, 메모, 태그 등을 포함한 구조화된 사건으로 변환

### 5) 기억 저장
- 원본 사진/영상 저장
- 구조화된 Story 저장
- 검색용 임베딩 저장

### 6) 기억 조회
- 새 입력이 들어오면 기존 장소/Story를 검색
- 정확 조회와 유사 조회를 병행

### 7) 판단
- 처음 방문인지
- 재방문인지
- 어떤 과거 기억을 보여줄지
- 추천/회상이 필요한지 결정

### 8) 응답 생성
- 과거 경험과 현재 맥락을 결합해 응답 생성
- 회상, 비교, 추천, 취향 분석 제공

### 9) 기억 갱신
- 이번 Story를 장기 기억에 반영
- 장소 프로필과 사용자 선호를 업데이트

## 7. 데이터 저장 및 조회 구조

### 저장 구조
이 프로젝트는 데이터를 3계층으로 나누어 저장한다.

#### 1. 원본 저장소
사진 및 영상 원본 보관
- 사진 파일
- 영상 파일
- OCR 원문
- 사용자 원문 메모

#### 2. 구조화 저장소
장면과 경험 정보를 정형화하여 저장
- 장소명
- 방문 시간
- 메뉴 목록
- 금액
- 감정/만족도
- 메모
- 태그
- 재방문 횟수

#### 3. 의미 검색 저장소
과거 Story를 의미 단위로 검색하기 위한 임베딩 저장
- Story 요약
- 장면 설명
- 장소 요약
- 사용자 반응 요약

### 조회 구조
새 입력이 들어오면 두 가지 방식으로 기억을 조회한다.

#### 1. 정확 조회
구조화 데이터베이스 기준
- 이 장소에 몇 번 왔는가
- 마지막 방문은 언제인가
- 지난번 메뉴는 무엇이었는가

#### 2. 유사 조회
벡터 검색 기준
- 비슷한 분위기의 장소
- 비슷한 상황에서 만족도가 높았던 경험
- 유사한 감정/메모 패턴

즉, 조회는 다음 조합으로 이루어진다.

**PostgreSQL 정확 조회 + Vector DB 유사 조회**

## 9. 기술 스펙

### 백엔드
- **FastAPI**: API 서버 및 요청 처리
- **Pydantic**: 입력/응답 스키마 검증

### AI/에이전트
- **OpenAI Vision**: 사진 및 영상 장면 해석
- **LLM**: Story 요약, 회상, 추천, 패턴 설명 생성
- **LangGraph**: 에이전트 흐름 제어

### 데이터 저장
- **PostgreSQL**: Scene, StoryEvent, PlaceProfile, UserPreferenceProfile 저장
- **pgvector 또는 Chroma**: Story 요약 및 Scene 설명 임베딩 저장
- **Object Storage (S3 계열)**: 원본 사진/영상 저장

### 핵심 모듈
- **Input Interpreter**: 사진/영상 입력 해석
- **Scene Builder**: Scene 묶음 구성
- **Story Builder**: StoryEvent 생성
- **Memory Store**: PostgreSQL/Vector DB 저장
- **Memory Retriever**: 정확 조회 및 유사 조회
- **Response Generator**: 회상/비교/추천 응답 생성

## 11. 프로젝트 핵심 가치

scene-story-agent의 핵심은 다음과 같다.

- 사진과 영상을 **장면(Scene)** 으로 해석한다.
- 장면들을 **이야기(Story)** 로 연결한다.
- 이야기를 장기 기억으로 저장한다.
- 이후 유사한 상황에서 과거 기억을 불러와 현재의 선택을 돕는다.

즉, 이 프로젝트는
**장면을 수집하는 서비스**가 아니라,
**장면을 기억하고 이야기로 엮어 다시 활용하는 AI 에이전트**이다.

---

## 12. 레포지토리명

**scene-story-agent**

### 이름 의미
- **scene**: 사용자가 현장에서 남긴 사진·영상 기반 장면
- **story**: 장면들을 시간축 위의 경험 서사로 재구성한 결과
- **agent**: 저장만 하지 않고 해석, 연결, 조회, 회상, 추천까지 수행하는 시스템

### 이름 선택 이유
- 식당/카페 방문에만 한정되지 않고 확장 가능함
- 사진뿐 아니라 영상까지 자연스럽게 포함 가능함
- 프로젝트의 핵심인 “장면을 이야기로 만든다”는 특성이 잘 드러남
