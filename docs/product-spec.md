## 2. 제품 정의

`scene-story-agent`는 사용자의 원본 기록을 그대로 보관하고, 사진 기반 AI 해석과 관련 기록 후보를 더해 기록 사이의 의미 연결을 보여주는 제품이다.

제품의 기본은 기록장이다. 사용자가 남긴 사진, 영상, 메모, 감정, 만족도는 원본 기록으로 저장되고 사용자는 나중에 그대로 다시 볼 수 있다.

AI는 원본 기록을 대체하지 않는다. AI는 사진을 해석하고 유사 기록, 관련 장소, 반복 주제, 타임라인 후보를 보여주는 보조 레이어로 동작한다.

기록이 쌓이면 제품은 개별 기록을 시간축 위에 이어 붙인다. 사용자는 장소, 주제, 기간을 기준으로 다시 생성된 타임라인형 콘텐츠를 볼 수 있다. 확장 단계에서는 특정 사람과의 기록도 별도 타임라인으로 만든다.

## 3. 제품 구조

제품은 다음 3개 층으로 구성한다.

### 원본 기록

사용자가 직접 남긴 정보다.

- 사진
- 영상
- 메모
- 감정
- 만족도
- 작성 시점
- 함께한 사람

### AI 해석 정보

원본 기록을 바탕으로 시스템이 생성한 정보다.

- 장면 유형
- 이미지 내 텍스트
- 장소 후보
- 방문 시간 후보
- 메뉴 후보
- 활동 후보
- 금액 후보
- 태그
- 요약
- 관련 기록 후보
- 재방문 후보
- 타임라인 후보

### 재생성 콘텐츠

원본 기록과 AI 해석 정보를 조합해 다시 보여주는 콘텐츠다.

- 시간순 타임라인
- 장소별 타임라인
- 비슷한 장면 묶음
- 반복 주제 묶음
- 기간별 회상 카드
- 특정인과의 타임라인

## 4. 제품 목표

MVP의 제품 목표는 다음이다.

- 사용자가 남긴 원본 기록을 그대로 다시 볼 수 있게 한다.
- 사진을 올리면 AI 해석 정보와 관련 기록 후보를 함께 보여준다.
- 기록이 쌓이면 시간축 위에서 이어지는 타임라인으로 보여준다.
- 같은 장소와 비슷한 장면의 기록을 다시 묶어 새로운 콘텐츠로 재생성한다.

## 5. 제품 판단 기준

제품 의사결정은 다음 기준으로 판단한다.

- 원본 기록을 훼손하지 않는다.
- 원본 기록과 AI 해석 정보를 구분해서 보여준다.
- AI 해석 정보는 사용자가 확인하고 고칠 수 있다.
- 저장된 기록이 타임라인과 연결 콘텐츠로 다시 쓰인다.
- 기록이 쌓일수록 재생성되는 콘텐츠 가치가 커진다.

## 6. 핵심 사용자

MVP의 핵심 사용자는 다음 사용자로 둔다.

- 외식, 카페, 여행, 전시처럼 장소 기반 경험을 자주 기록하는 사용자
- 사진은 많이 찍지만 나중에 다시 찾거나 정리하지 못하는 사용자
- 짧은 메모로 당시 감정, 만족도, 선택 이유를 남기고 싶은 사용자
- 과거 기록을 시간축, 장소, 주제 기준으로 다시 보고 싶은 사용자

## 7. 사용자 문제

MVP에서 해결할 문제는 다음으로 한정한다.

- 사진만으로는 당시 맥락을 떠올리기 어렵다.
- 사진과 메모는 남아 있지만 서로 이어진 기록으로 보기 어렵다.
- 같은 장소나 비슷한 상황의 기록을 바로 확인하기 어렵다.
- 기록이 쌓여도 새로운 회상 콘텐츠로 재생성되지 않는다.

## 8. 제품 가설

MVP는 다음 가설을 검증한다.

- 원본 기록을 그대로 보면서 AI 해석 정보를 함께 제공하면 사용자는 기록을 더 쉽게 찾는다.
- 사진 기반 관련 기록 후보를 보여주면 사용자는 비슷한 과거 기록을 발견한다.
- 기록이 3개 이상 쌓이면 사용자는 시간축으로 이어진 회상 콘텐츠를 기대한다.
- 기록을 장소와 주제 기준으로 묶어 보여주면 사용자는 개인화된 타임라인 콘텐츠로 인식한다.

## 9. 핵심 가치 제안

사용자에게 제공할 가치는 다음이다.

- 원본 기록을 그대로 보관하고 다시 보여준다.
- 사진을 올리면 AI가 해석한 장면 정보와 관련 기록을 보여준다.
- 기록이 쌓이면 시간축으로 이어진 타임라인을 만든다.
- 장소, 주제, 기간 기준으로 기록을 다시 묶어 새로운 콘텐츠로 보여준다.

## 10. 제품 원칙

MVP 화면과 기능은 다음 원칙을 따른다.

- 원본 기록은 사용자가 남긴 그대로 보존한다.
- AI 해석 정보는 원본 기록을 보조한다.
- 사용자는 원본 기록과 AI 해석 정보를 구분해서 본다.
- AI가 추출한 값은 사용자가 수정할 수 있다.
- 관련 기록 계산 결과는 원시 값이 아니라 유사 기록, 연결 후보, 태그, 타임라인 형태로 보여준다.
- 추천보다 기록 회상과 타임라인 생성을 먼저 제공한다.
- 자동화 결과보다 사용자의 최종 확인을 우선한다.

## 11. 핵심 사용 흐름

### 첫 기록

1. 사용자가 사진을 업로드한다.
2. 사용자가 짧은 메모, 감정, 만족도를 선택 입력한다.
3. 시스템이 사진, 메모, 감정, 만족도를 원본 기록으로 저장한다.
4. AI가 장소, 방문 시간, 장면 유형, 텍스트, 메뉴, 활동, 금액, 유사 기록, 재방문, 타임라인 후보를 만든다.
5. 사용자가 원본 기록과 AI 해석 정보를 함께 확인한다.
6. 시스템이 기록 상세와 관련 기록 후보를 보여준다.

### 재방문

1. 사용자가 새 사진을 업로드한다.
2. 시스템이 원본 기록을 저장하고 기존 기록과 비교한다.
3. 같은 장소 또는 유사 경험 후보를 보여준다.
4. 사용자가 현재 기록과 연결할지 선택한다.
5. 시스템이 이전 기록 요약과 이번 기록을 함께 보여준다.
6. 저장 후 방문 횟수와 변화 내용을 갱신한다.

### 기록 회상

1. 사용자가 장소, 주제, 감정, 기간 중 하나로 기록을 찾는다.
2. 시스템이 정확한 기록과 유사한 기록을 함께 보여준다.
3. 사용자가 기록 상세를 연다.
4. 시스템이 원본 기록, AI 해석 정보, 연결 기록을 함께 보여준다.

### 타임라인 생성

1. 사용자가 기록을 계속 추가한다.
2. 시스템이 시간, 장소, 장면 유형 기준으로 기록을 연결한다.
3. 사용자가 타임라인 기준을 선택한다.
4. 시스템이 연결된 기록을 시간순 콘텐츠로 보여준다.
5. 사용자는 타임라인에서 원본 기록과 AI 해석 정보를 함께 확인한다.

## 12. 대표 사용 시나리오

### 첫 방문 기록

사용자는 식당, 카페, 여행지, 전시 공간에서 사진을 남긴다. 제품은 사용자가 남긴 사진과 메모를 원본 기록으로 저장한다. 동시에 간판, 메뉴판, 음식, 영수증, 공간 분위기를 해석하고 사용자가 확인할 수 있는 AI 해석 정보를 만든다.

AI 해석 정보에는 다음 후보가 포함된다.

- 장소
- 방문 시간
- 메뉴
- 활동
- 금액
- 장면 유형
- 태그
- 요약
- 유사 기록
- 재방문
- 타임라인

### 재방문 인식

사용자가 같은 장소나 비슷한 장소에서 새 사진을 남기면 제품은 과거 기록 후보를 보여준다. 사용자는 현재 기록을 기존 장소 기록과 연결하거나 새 기록으로 저장한다.

재방문 응답은 다음 정보를 우선 제공한다.

- 이전 방문 횟수
- 마지막 방문 시점
- 지난 선택
- 당시 메모
- 당시 만족도
- 연결 근거

### 타임라인 콘텐츠 재생성

기록이 누적되면 제품은 기록을 새 콘텐츠로 재구성한다. 사용자는 특정 장소의 변화, 특정 기간의 활동, 반복되는 주제를 타임라인으로 다시 볼 수 있다.

타임라인 콘텐츠는 다음 기준으로 생성된다.

- 시간
- 장소
- 장면 유형
- 감정
- 반복 주제

## 13. 핵심 제품 루프

MVP의 제품 루프는 다음과 같다.

1. 사진 업로드
2. 원본 기록 저장
3. AI 해석 및 관련 기록 후보 생성
4. 사용자 확인
5. 기록 상세 조회
6. 유사 기록 연결
7. 타임라인 콘텐츠 생성
8. 새 기록 추가

이 루프가 반복될수록 제품의 개인화 가치가 커진다.

## 18. 품질 기준

MVP 출시 기준은 다음이다.

- 사용자가 원본 사진과 메모를 그대로 다시 볼 수 있다.
- 사진 업로드 후 AI 해석 결과가 표시된다.
- 사용자가 AI 추출 결과를 수정할 수 있다.
- 기록 저장 후 목록과 상세에서 다시 확인할 수 있다.
- 같은 장소 후보가 있으면 재방문 후보로 표시된다.
- 누적 기록을 시간순으로 볼 수 있다.
- 분석 실패 시 사용자가 직접 기록을 작성할 수 있다.
- 저장 실패 시 사용자가 입력값을 잃지 않는다.

- 팔로우, 댓글, 좋아요

- 자동 인물 인식 기반 사람 태깅
- 특정인과의 타임라인
