# 개인정보 보호 조치

## 1. 문서 목적

이 문서는 `scene-story-agent`가 사진, 영상, 메모, 위치성 정보, AI 해석 정보를 처리할 때 필요한 개인정보 보호 조치를 정리한다.

이 문서는 제품·기술 설계용 초안이다. 출시 전 개인정보 보호 전문가 또는 법률 전문가 검토를 거친다.

## 2. 적용 전제

초기 서비스는 한국 이용자를 대상으로 한다.

MVP는 다음 범위로 개인정보를 처리한다.

- 사용자가 업로드한 사진
- 사용자가 업로드한 영상
- 사용자가 입력한 메모
- 감정과 만족도
- 기록 작성 시점
- 사진에서 추출한 텍스트
- 장소 후보
- 장면 태그
- AI 요약
- 임베딩 벡터
- 서비스 계정 정보

MVP는 다음 처리를 하지 않는다.

- 실시간 위치 추적
- 자동 인물 인식
- 얼굴 식별
- 생체 인증
- 결제 정보 자동 수집
- 공개 피드 제공

## 3. 개인정보 분류

서비스 데이터는 다음 기준으로 분류한다.

| 구분 | 예시 | 처리 기준 |
|---|---|---|
| 계정 정보 | 이메일, 닉네임, 로그인 식별자 | 회원 관리 목적에 한정한다. |
| 원본 기록 | 사진, 영상, 메모, 감정, 만족도 | 사용자가 직접 조회·삭제할 수 있게 한다. |
| 이미지 내 개인정보 | 얼굴, 차량번호, 영수증 일부 정보, 간판 주변 인물 | 원본 기록에 포함될 수 있으므로 보호 대상에 포함한다. |
| 위치성 정보 | 장소명, 방문 시점, EXIF 위치값 | 개인의 동선 추정 가능성이 있으므로 별도 관리한다. |
| AI 해석·발췌 정보 | 장면 유형, OCR 텍스트, 요약, 태그 | 원본 기록과 연결된 개인정보로 관리한다. |
| 벡터 정보 | 이미지·기록 임베딩 | 원본 기록과 연결 가능하므로 개인정보에 준해 관리한다. |
| 운영 로그 | 접속 기록, 오류 로그, AI 호출 로그 | 최소 항목만 남기고 보관 기간을 제한한다. |

## 4. 수집 원칙

개인정보 수집은 다음 원칙을 따른다.

- 서비스 제공에 필요한 항목만 수집한다.
- 필수 항목과 선택 항목을 구분한다.
- 수집 목적과 보유 기간을 수집 시점에 안내한다.
- 사진 원본과 AI 해석 정보를 구분해 저장한다.
- AI 학습 목적의 사용은 기본 서비스 목적과 분리한다.
- 사용자가 원본 기록을 삭제하면 연결된 AI 해석·발췌 정보와 벡터 정보도 함께 삭제한다.

## 5. 동의 구조

서비스 가입 또는 첫 기록 생성 전에 다음 동의를 분리한다.

### 필수 동의

- 서비스 이용을 위한 계정 정보 처리
- 사진, 영상, 메모의 저장과 조회
- 사진 기반 AI 해석 정보 생성
- 유사 기록 검색을 위한 임베딩 생성

### 선택 동의

- 마케팅 알림 수신
- 품질 개선 목적의 분석
- AI 모델 개선 목적의 데이터 활용

### 별도 동의 검토 대상

- 민감정보 가능성이 있는 메모 처리
- 위치정보 처리
- 국외 이전 또는 국외 처리위탁
- 만 14세 미만 아동의 개인정보 처리

## 6. 개인정보 처리방침 필수 항목

개인정보 처리방침에는 다음 항목을 포함한다.

- 개인정보 처리 목적
- 처리하는 개인정보 항목
- 개인정보 보유 및 이용 기간
- 개인정보 파기 절차와 방법
- 개인정보 제3자 제공 여부
- 개인정보 처리업무 위탁 여부
- 개인정보 국외 이전 여부
- 정보주체와 법정대리인의 권리와 행사 방법
- 개인정보 안전성 확보 조치
- 자동 수집 장치 사용 여부
- AI 자동화 처리 또는 자동화된 결정 여부
- 가명정보 처리 여부
- 개인정보 보호책임자와 문의 창구
- 권익침해 구제 방법
- 처리방침 변경 이력

## 7. 사진·영상 처리 기준

사진과 영상은 원본 기록이지만 개인정보가 포함될 수 있다.

다음 조치를 적용한다.

- 원본 파일 접근 권한을 사용자 본인으로 제한한다.
- 운영자 직접 접근은 장애 대응과 법적 요청 대응으로 제한한다.
- 운영자 접근 시 접근 사유와 시간을 로그로 남긴다.
- 얼굴 식별, 인물명 추론, 자동 인물 태깅은 MVP에서 제공하지 않는다.
- 이미지 내 제3자 얼굴이 포함될 수 있음을 처리방침에 안내한다.
- 사용자가 원본 파일을 삭제하면 연결된 AI 해석·발췌 정보와 벡터 정보도 함께 삭제한다.

## 8. 위치성 정보 처리 기준

MVP는 실시간 위치 추적을 하지 않는다.

다만 사진에서 장소가 추론되거나 EXIF에 위치값이 포함될 수 있으므로 다음 기준을 둔다.

- EXIF 위치값 저장 여부를 제품 정책으로 명확히 정한다.
- EXIF 위치값을 저장하지 않는 경우 업로드 시 제거한다.
- EXIF 위치값을 저장하는 경우 수집 목적과 보유 기간을 별도 고지한다.
- 장소명과 방문 시점 조합은 개인 동선 추정 가능 정보로 관리한다.
- 위치기반서비스에 해당하는 기능을 추가하기 전 위치정보법 검토를 진행한다.

## 9. AI 처리 기준

AI 처리에는 다음 기준을 적용한다.

- AI 해석·발췌 정보는 원본 기록의 보조 정보로 표시한다.
- 사용자는 AI 해석·발췌 정보를 수정하거나 삭제할 수 있다.
- AI 해석 실패 시 수동 기록 경로를 제공한다.
- AI 모델 제공자에게 전송되는 데이터 항목을 처리방침에 공개한다.
- AI 모델 제공자가 국외 사업자인 경우 국외 이전 또는 국외 처리위탁 여부를 검토한다.
- AI 모델 개선 목적으로 사용자 원본 기록을 사용하려면 별도 선택 동의를 받는다.
- 자동화된 결정으로 이용자 권리나 의무에 중대한 영향을 주는 기능은 MVP에서 제공하지 않는다.

## 10. 벡터 정보 처리 기준

임베딩 벡터는 원본 기록과 연결될 수 있으므로 개인정보에 준해 관리한다.

다음 조치를 적용한다.

- 벡터 저장소에는 사용자 식별자를 직접 저장하지 않는다.
- 벡터와 원본 기록의 연결 키는 별도 테이블에서 관리한다.
- 원본 기록 삭제 시 연결된 벡터를 삭제한다.
- 벡터만으로 외부 공유하지 않는다.
- 벡터 데이터 백업 보관 기간을 제한한다.

## 11. 보관 기간과 삭제

보관 기간은 다음 기준으로 정한다.

| 데이터 | 보관 기간 초안 | 삭제 기준 |
|---|---|---|
| 계정 정보 | 회원 탈퇴 시까지 | 탈퇴 요청 시 삭제 |
| 원본 사진·영상 | 사용자가 삭제할 때까지 | 기록 삭제 시 삭제 |
| 메모·감정·만족도 | 사용자가 삭제할 때까지 | 기록 삭제 시 삭제 |
| AI 해석·발췌 정보 | 원본 기록 보관 기간과 동일 | 원본 기록 삭제 시 삭제 |
| 임베딩 벡터 | 원본 기록 보관 기간과 동일 | 원본 기록 삭제 시 삭제 |
| 접속 로그 | 보안 목적 기간 동안 | 보관 기간 만료 시 삭제 |
| AI 호출 로그 | 장애 대응 목적 기간 동안 | 보관 기간 만료 시 삭제 |

정확한 보관 기간은 출시 전 법률 검토와 운영 정책으로 확정한다.

## 12. 정보주체 권리

사용자는 다음 권리를 행사할 수 있어야 한다.

- 개인정보 열람
- 개인정보 정정
- 개인정보 삭제
- 개인정보 처리정지
- 동의 철회
- 기록 내보내기
- AI 해석·발췌 정보 삭제
- 회원 탈퇴

서비스 화면에는 최소한 다음 기능을 제공한다.

- 기록 단위 삭제
- 전체 기록 삭제
- 계정 삭제
- 개인정보 문의

## 13. 안전성 확보 조치

초기 구현부터 다음 조치를 적용한다.

- 전송 구간 암호화
- 저장 파일 접근 제어
- 원본 파일 비공개 저장
- 운영자 접근 권한 최소화
- 운영자 접근 로그 기록
- DB 접근 권한 분리
- 백업 데이터 암호화
- 개인정보 포함 로그 마스킹
- API 인증과 권한 검사
- 삭제 요청 처리 로그 보관
- AI 호출 데이터 최소화

### 접근 제어 기준

서비스 접근은 다음 기준을 따른다.

- 클라이언트는 원본 파일 저장소에 직접 쓰지 않는다.
- 클라이언트는 API 서버를 통해서만 기록을 생성한다.
- API 서버는 사용자 인증 후 원본 파일 접근 URL을 발급한다.
- 원본 파일 저장소는 기본 비공개로 둔다.
- DB 서비스 키는 서버에서만 사용한다.
- AI API 키와 DB 접속 정보는 코드에 저장하지 않는다.
- 운영자 DB 접근은 최소 인원으로 제한한다.
- AI 호출 로그에는 원본 이미지 URL을 그대로 저장하지 않는다.

## 14. 제3자 제공과 처리위탁

다음 구분을 명확히 한다.

### 제3자 제공

사용자의 개인정보를 다른 회사가 자신의 목적을 위해 쓰는 경우다.

MVP에서는 제3자 제공을 하지 않는다.

### 처리위탁

서비스 제공을 위해 외부 사업자가 저장, 분석, 전송 업무를 대신 처리하는 경우다.

MVP에서 검토할 처리위탁 대상은 다음이다.

- 클라우드 인프라
- S3 호환 오브젝트 스토리지
- AI 모델 API 제공자
- 이메일 또는 알림 발송 서비스
- 로그·모니터링 서비스

처리위탁이 있으면 처리방침에 수탁자와 위탁 업무를 공개한다.

## 15. 국외 이전 검토

AI 모델 API, 클라우드, 로그 서비스가 국외에서 데이터를 처리할 수 있다.

국외 이전 또는 국외 처리가 발생하면 다음 항목을 확인한다.

- 이전되는 개인정보 항목
- 이전 국가
- 이전 일시와 방법
- 이전받는 자
- 이용 목적
- 보유 및 이용 기간
- 거부 방법과 거부 시 영향
- 보호 조치

## 16. 유출 대응

개인정보 유출 또는 의심 상황이 발생하면 다음 절차를 따른다.

1. 유출 의심 사실을 기록한다.
2. 유출 범위와 영향을 확인한다.
3. 추가 유출을 차단한다.
4. 정보주체 통지 필요성을 판단한다.
5. 관계기관 신고 필요성을 판단한다.
6. 재발 방지 조치를 기록한다.

유출 대응 기준은 개인정보 보호법과 관련 시행령 기준으로 별도 운영 절차를 작성한다.

## 17. 출시 전 필수 점검

출시 전 다음 항목을 완료한다.

- 개인정보 처리방침 작성
- 개인정보 수집·이용 동의 화면 작성
- 선택 동의 분리
- 처리위탁 목록 작성
- 국외 이전 여부 확인
- EXIF 위치값 처리 정책 확정
- 원본 기록 삭제 시 파생 데이터 삭제 검증
- 원본 파일 저장소 비공개 접근 검증
- DB 접근 권한과 서비스 키 사용 범위 검증
- AI 호출 로그의 개인정보 마스킹 검증
- 운영자 접근 권한 정책 작성
- 개인정보 보호책임자 지정
- 유출 대응 절차 작성
- AI 모델 제공자 데이터 보관 정책 확인
